无痛学习

Posted on 2024-03-17

关键词：兴趣，刷题

想考雅思其实已经很久了，但是一直没有完成这个目标。
从否定一直以来突击的学习方法，到转变为兴趣驱动慢慢积累，再到刻意练习，学习方法一直在变，但是却没有实践结果予以佐证。因为没有成果，所以一直在质疑和转变学习方法，没有进入一个有效的正循环。
想要拿到结果，方法固然重要，但是明确的目标和持续的努力更加重要。通过一种方式或者多种方式的结合拿到结果后，再持续迭代，才能够越来越高效无痛的拿到结果。
我想这个正循环可以从考雅思开始。思考也从这里开始。

  1. 动力。优先级低，动力不足。虽然我羡慕别人英语说的好，能阅读英文文档，但是这并不是我生存需要的核心技能，只是我渴望并希望拿来炫耀自己的饰品，所以优先级总是比其他生存技能低。
  2. 心态。一旦开始学习，就会自动开始竞争模式，担心比别人学的慢，比别人学的差，怕考试成绩不理想，这是从小养成的学习心态。这种心态，压力大但效率高，面向考试和分数。但是曾几何时，我否定了自己曾经的努力，认为分数和能力不成正比。所以开始转变心态，以学到就是赚到，不管别人会不会，我会了就行安慰自己，期望有能力自然有分数。这种心态确实安慰了焦虑抑郁期间的自己，学习成果还是有的，只是不多。其实我的目标还是拿到结果，这是骗不了自己的，光是哄着自己开心了，没有拿到结果，也还是会反噬自己。
  3. 能力。现在专注力和学习能力都没有以前好了。也不是自己能力变差了，只是要思考的事情越来越多，专注在一件事情上的本领变弱了。而且确实身体没有以前好，学习时间比较有限。

总结下来，之所以学习进程如此艰难，是因为学习成本太高了。我把学习看的太重要了，不仅是学习的结果甚至是学习的过程。所以学习之前要做心理准备，要固定几点学多长时间休息多长时间，按照什么路径去学习，又要想办法让学习的过程尽可能快乐，又要想办法学习各种学习的能力和方法，又会想到学习会伤害我的腰和眼睛，又想要要学习了有没有给自己补充足够的营养，又会想其他人学的怎么样了，又会想学不会怎么办，一边学习一边批评自己，想学的时候怕自己伤害自己的身体，阻止自己，不想学的时候觉得自己懒惰但又觉得应该听从自己内心的安排，羞耻并忐忑不安的阻止自己。写下来，感觉自己真的是个戏精。
所以，要提升自己的学习能力，就要先把学习成本降下来。
解决方案：把学习当成一件普通的事，必须要做的事，跟上班一样，跟吃饭睡觉一样，跟活着一样。不需要再去纠结什么想不想，有什么意义，为了什么目的，必须做。不要把学习跟太多事情关联在一起，给自己设不必要的限制。不要把方法看的太重要，痛苦快乐都行。不要把结果看的太重要，学的时候认真专注就好，产生副作用再单独解决。
如果学习变成一个不痛不痒，不影响正常生活的事情，开始的门槛就降低了。
过程也很简单，调研梳理学习路径，拆解任务，逐个执行，定期复盘。
重点是，调研的时候只想调研，拆任务的时候只想任务，执行的时候只执行。每个过程都不要反刍，不要质疑自己，只有复盘的时候做。
再有就是过程中，可以休息，但不可以放弃。我想如果无痛学习，也就没什么好放弃不放弃的，做不做都一样。
希望自己可以早日掌握“无痛学习”这项技能。以及“无痛”做任何事情。
总结无痛学习的一些关键词：
什么时间做都行，进行多长时间都行，不要给自己加戏，长期坚持。

专注

Posted on 2024-03-14

刚刚写写音标，突然脑海里出现了其他的事情，让我没有办法专注。当然现在我已经忘记当时脑海里出现的是什么念头了。
所以我想说说，为什么我无法专注的做一件事情。
首先，这不能责怪自己，或者觉得自己有毛病。这是正常的生理现象，只不过我的想法比较多，所以想法出现的也会比较频繁。
我想说的是，无法专注于一件事情，大概是因为这件事情对我来说优先级不是最高的，或者有很多优先级高的事情在等着我解决。
换句话说，我还是没有想清楚什么事情对我是最重要的。
至今我都很矛盾，我认为身心健康是最重要的，但是当我为了自己的身心健康拒绝加班的时候，我会害怕；我认为自由是最重要的，但是当我为了自由而拒绝别人的控制时，我会恐慌。甚至觉得自己是在以身心健康和自由为理由，放纵自己。
我没有清晰的认识自己接纳自己，而是不断的抵抗和怨恨无能的那部分自己，却不愿意接受自己。
得到一些就会失去一些，
你为了健康不加班，别人加班获得了好的绩效就要接受。
你为了省时间不去参加会议，不去向上管理，其他人因此获利就要接受。
你不愿意相亲结婚，与他人共担生活的责任，享受不到家庭的温暖和孩童的快乐就要接受。
因为你获得了更多自由支配的时间，你可以利用这些时间去做任何你想做的事情。
与此同时，你又想，我有这些时间不也是无用，没做什么事情，还不如用来参与竞争；你又想，其他人不也没怎么加班，不仅有时间还有我有的一切。
但这些只是你的想象，事实并不如此，你不知道其他人的感受，也没必要跟其他人去比较。
你只需要让自己好起来。而且我相信，不管我努力还是不努力，结果都是一样的，我只会干更多的活儿受更多的累，不一定还会坚持不下去，挣更少的钱。
说回来，我为何无法专注，如何让自己专注。既然是因为优先级不明确，就明确优先级。
优先级1: 身心健康。略有不适就适当休息，不必焦虑。
优先级2: 强健体魄。享受阳关，适度锻炼。
优先级3: 提升自我。为兴趣学习，不为目的学习。
优先级4: 开拓视野。看看外面的世界。
当然，小念头乱跑是正常的生理现象，要接受，并通过锻炼去缓解不适。在书里学到过一种方法：当念头出现，不去理会，记录下来，继续专注。
还有一种方法：做过的事情，做到自己接受的程度，不然不仅会担惊受怕，还容易返工。要稳不要急。

递归转循环

l1 = [1,2,3]
l2 = [4,5]
l3 = [6,7,8]
l = [l1, l2, l3]
def comb(l):
if len(l)==0:
return l
l0 = [[]]
while len(l)>0:
l1 = l[0]
l = l[1:]
res = []
for i in l0:
for j in l1:
res.append(i+[j])
l0 = res
return l0, len(l0)
print(comb(l))

def combi(seq):
if not seq:
yield []
else:
for element in seq[0]:
for rest in combi(seq[1:]):
yield [element] + rest

al - data structure design

Posted on 2022-11-15 Edited on 2022-11-16

LRU缓存: map+双向链表

最近最少使用
功能：

  1. 以 “容量” 初始化缓存
  2. get(key)如果key在缓存中，返回关键字的值，否则-1， O(1)
  3. put(key,value)如果在，更新，如果不在，插入，如果插入时没有容量剩余，删除最久没有使用的，O(1)

对数据结构的要求：

  1. fifo 队列 (不行，删除节点不能满足O(1)), 需要用双向链表
  2. O(1)查找 - map
    [key1:node1, key2:node2, key3:node3]
    node2->node1->node3

函数：
查找：哈希查找
新增：在开头新增，哈希增加
删除：删除某个节点，节点删除，哈希删除
put: 如果在，删除已有节点，size–；如果size==capicity，删除末尾节点，size–；新增

FRU缓存 - 堆

O(1) 时间插入、删除和获取随机元素: 变长数组+map

功能：
insert(val) 如果不存在，插入 - map
remove(val) 如果存在，删除 - map
getRandom() 概率相等的返回随机元素 - 列表

难点：

  1. 如何O(1)在数组中删除一个元素，并不改变其他元素的下标
    答案：要删除元素与最后一个元素做交换即可
  2. 如何实现随机
    答案：rand.Intn(size)

全 O(1) 的数据结构

inc(string) 字符串计数+1，存在+1，不存在插入记为1 - map
dec(string) 字符串计数-1，减后为0，删除 - map
getMaxKey() 返回计数最大的，没有“” - 最大栈，因为涉及O(1)删除任意值，所以用双向列表
getMinKey() 返回计数最小的，没有“” - 最小栈

官方题解给的思路：
构造一个递增的双向链表尾
inc: 如果+1的后一个节点有，加入后一个节点；如果没有或尾，新增一个
dec: 如果-1的前一个节点有，加入；如果没有且不是头，新增；如果是头，删除
getMaxKey(): 返回尾
getMinKey(): 返回头

最大栈

MaxStack() 初始化栈对象
void push(int x) 将元素 x 压入栈中。
int pop() 移除栈顶元素并返回这个元素。
int top() 返回栈顶元素，无需移除。
int peekMax() 检索并返回栈中最大元素，无需移除。
int popMax() 检索并返回栈中最大元素，并将其移除。如果有多个最大元素，只要移除 最靠近栈顶 的那个。

思路：

  1. 用两个栈，最大栈元素为值和索引，popMax时，将索引之后的重新入栈
  2. 用一个最大栈链表

树状数组

树状数组是一种可以动态维护序列前缀和的数据结构（序列下标从 1 开始），它的功能是：
单点修改add(index, val): 第index个数增加val； O(logN)
区间查询prefixSum(index)：查询前 index 个元素的前缀和。O(logN)

lowbit

非负整数n在二进制表示下，最低位1以及后面的0构成的数值。
计算方法：x&-x (与取反加1后的值进行按位与)（补码）
lowbit(x)可以代表节点的层级，也就是t[x]覆盖的数组长度
add()操作持续找每一个index的上一层，+lowbit()
prefixSum()操作找左上的每个点，-lowbit()

KMP算法

字符串匹配算法，构造next数组，使后缀前缀相同，使得对主串的遍历只需要一次，时间复杂度从O(nm)降低到O(n+m)
tips:

  1. next数组都初始化为-1
  2. next数组从1到m-1遍历
  3. kmp从0到n遍历
