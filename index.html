认识自己的方法

理论：

  • 约哈里窗
    方式：
  • 他人视角：“认知多维镜”
  • 自我观察：“局外人”视角

    约哈里窗

    “自我意识的发现-反馈模型”
    四个窗口：
    自己知道，自己未知
    他人知道，他人未知

自己知道，他人知道：开放我 - 外显个性
自己知道，他人未知：隐藏我 - 个人癖好，童年记忆，痛苦经验
自己未知，他人知道：盲目我 - 优缺点，习惯性动作，某些对人处事的态度
自己未知，他人未知：未知我 - 潜能，天赋

我们应该将开放我最大化，这样能够对自我认识更加透彻，也能增进他人对自己的理解，让自己内外一致，活的更加坦荡，更容易被人接受。

我们要尽可能的展示自己的优势，同时不要惧怕暴露自己的一些问题。只有人们展示信息足够多和充分时，人们才能接受到来自他人准确的反馈。

我们遇见所有人，都是为了遇见自己，一个更完整的自己。

避免和外界过多的接触，开放我就会小，不认识自己，就会产生内外冲突感。
所以要更加主动的尝试一些方法：
分享自己，倾听他人，展开讨论

360信息反馈 - 认知多维镜

方法：

  1. 开放自我，坦诚交友 - 收集反馈，减少盲目我
  2. 建立信任 - 开放隐藏我

通过自我坦诚与反馈，逐步缩小盲目我和隐藏我，使开放我更加广阔。
在他人说话的时候不要打断，也不要急于解释，认真倾听才能得到更多有价值的反馈。
需要问题问题是：缺点，优点，性格特点，具体事例。

AE在wrk中的应用

wrk开启压测的流程

主要参数：线程数，连接数，压测时间，超时时间，请求头，压测脚本
主要对象：cfg, sock, statistics, http_parse_settings
线程对象属性：eventloop, 连接数，lua虚拟机

  1. 解析参数
  2. 定义 schema, host, port, service
  3. 如果schema解析为https，进行ssl连接
  4. 设置忽略 SIGINT（interrupt 中断信号）, SIGPIPE （管道破裂信号）， 否则会导致压测服务退出
  5. 根据压测脚本初始化全局的lua虚拟机
  6. 进行地址解析和socket连接，测试host是否可连接
  7. 根据设置的线程数，创建线程对象，执行线程任务
  8. 睡眠等待压测时间
  9. 阻塞等待所有线程结束
  10. 统计结果输出

每个线程执行的任务

  1. 根据每个线程需要启动的连接数，建立socket连接
  2. 根据eventloop建立定时事件，每100ms统计一次数据
  3. 开始处理eventloop
  4. 删除eventloop

建立socket连接做了什么

  1. 建立fd
  2. 设置为O_NONBLOCK
  3. 开启连接
  4. 创建一个文件事件

文件事件中做了什么

  1. 查看socket连接状态
  2. 初始化httpparser
  3. 创建两个文件事件，fd可读事件，fd可写事件

fd可写事件

超时或者写完，删除可写事件

fd可读事件

执行http_parser_execute发送请求，等待结果返回，读响应结果，回调response函数

AE主要函数

loop = aeCreateEventLoop(10 + cfg.connections * 3)
aeCreateTimeEvent(loop, RECORD_INTERVAL_MS, record_rate, thread, NULL)
aeCreateFileEvent(loop, fd, flags, socket_connected, c)

aeDeleteEventLoop(loop)

cd /usr/local/mysql/
index文件： mysql-bin.index
binlog文件：mysql-bin.000055
redolog文件：ib_logfile0
undolog文件：undo_001

查看binlog

../bin/mysqlbinlog –no-defaults mysql-bin.000055

sql -> show binlog events in ‘mysql-bin.000055’;
sql -> show variables like ‘binlog_format’; // statement or row
sql -> show variables like ‘log_%’;
sql -> show master status\G;

查询 BINLOG 格式

show VARIABLES like ‘binlog_format’;

查询 BINLOG 位置

show VARIABLES like ‘datadir’;

查询当前数据库中 BINLOG 名称及大小

show binary logs;

查看 master 正在写入的 BINLOG 信息

show master status\G;

通过 offset 查看 BINLOG 信息

show BINLOG events in ‘mysql-bin.000034’ limit 9000, 10;

通过 position 查看 binlog 信息

show BINLOG events in ‘mysql-bin.000034’ from 1742635 limit 10;

查询 binlog

mysqlbinlog –no-defaults –base64-output=decode-rows -v
–start-datetime “2021-11-09 20:00:00”
–database testdb mysql-bin.000055| less

导出 BINLOG

mysqlbinlog –no-defaults –base64-output=decode-rows -v
–start-datetime “2021-11-09 20:00:00”
–database testdb mysql-bin.000055 > binlog_raw.sql

恢复

通过 BINLOG 进行恢复。

../bin/mysqlbinlog –no-defaults –start-position=482 –stop-position=1327 –database=testdb mysql-bin.000055 | mysql -u username -p password

通过 BINLOG 导出的 sql 进行恢复。

mysql -u username -p password < binlog_raw.sql
