但是如果在列表数量不确定的情况下，如何实现一个通用的函数呢？这时我想到了递归。

递归转循环

l1 = [1,2,3]
l2 = [4,5]
l3 = [6,7,8]
l = [l1, l2, l3]
def comb(l):
if len(l)==0:
return l
l0 = [[]]
while len(l)>0:
l1 = l[0]
l = l[1:]
res = []
for i in l0:
for j in l1:
res.append(i+[j])
l0 = res
return l0, len(l0)
print(comb(l))

def combi(seq):
if not seq:
yield []
else:
for element in seq[0]:
for rest in combi(seq[1:]):
yield [element] + rest

LRU缓存: map+双向链表

最近最少使用
功能：

  1. 以 “容量” 初始化缓存
  2. get(key)如果key在缓存中，返回关键字的值，否则-1， O(1)
  3. put(key,value)如果在，更新，如果不在，插入，如果插入时没有容量剩余，删除最久没有使用的，O(1)

对数据结构的要求：

  1. fifo 队列 (不行，删除节点不能满足O(1)), 需要用双向链表
  2. O(1)查找 - map
    [key1:node1, key2:node2, key3:node3]
    node2->node1->node3

函数：
查找：哈希查找
新增：在开头新增，哈希增加
删除：删除某个节点，节点删除，哈希删除
put: 如果在，删除已有节点，size–；如果size==capicity，删除末尾节点，size–；新增

FRU缓存 - 堆

O(1) 时间插入、删除和获取随机元素: 变长数组+map

功能：
insert(val) 如果不存在，插入 - map
remove(val) 如果存在，删除 - map
getRandom() 概率相等的返回随机元素 - 列表

难点：

  1. 如何O(1)在数组中删除一个元素，并不改变其他元素的下标
    答案：要删除元素与最后一个元素做交换即可
  2. 如何实现随机
    答案：rand.Intn(size)

全 O(1) 的数据结构

inc(string) 字符串计数+1，存在+1，不存在插入记为1 - map
dec(string) 字符串计数-1，减后为0，删除 - map
getMaxKey() 返回计数最大的，没有“” - 最大栈，因为涉及O(1)删除任意值，所以用双向列表
getMinKey() 返回计数最小的，没有“” - 最小栈

官方题解给的思路：
构造一个递增的双向链表尾
inc: 如果+1的后一个节点有，加入后一个节点；如果没有或尾，新增一个
dec: 如果-1的前一个节点有，加入；如果没有且不是头，新增；如果是头，删除
getMaxKey(): 返回尾
getMinKey(): 返回头

最大栈

MaxStack() 初始化栈对象
void push(int x) 将元素 x 压入栈中。
int pop() 移除栈顶元素并返回这个元素。
int top() 返回栈顶元素，无需移除。
int peekMax() 检索并返回栈中最大元素，无需移除。
int popMax() 检索并返回栈中最大元素，并将其移除。如果有多个最大元素，只要移除 最靠近栈顶 的那个。

思路：

  1. 用两个栈，最大栈元素为值和索引，popMax时，将索引之后的重新入栈
  2. 用一个最大栈链表

线段树

每一个节点，存一个区间和
根结点是所有数的和

树状数组

树状数组是一种可以动态维护序列前缀和的数据结构（序列下标从 1 开始），它的功能是：
单点修改add(index, val): 第index个数增加val； O(logN)
区间查询prefixSum(index)：查询前 index 个元素的前缀和。O(logN)

lowbit

非负整数n在二进制表示下，最低位1以及后面的0构成的数值。
计算方法：x&-x (与取反加1后的值进行按位与)（补码）
lowbit(x)可以代表节点的层级，也就是t[x]覆盖的数组长度
add()操作持续找每一个index的上一层，+lowbit()
prefixSum()操作找左上的每个点，-lowbit()

KMP算法

字符串匹配算法，构造next数组，使后缀前缀相同，使得对主串的遍历只需要一次，时间复杂度从O(nm)降低到O(n+m)

  2. next数组从1到m-1遍历
  3. kmp从0到n遍历
