int

go 中的 int 是 int 64

由上，因为a是int64，所以不同存储空间间隔8bytes=64bits
不仅如此还可以证明，在go语言中，参数传递，传递的是“值”，而不是引用。
“值” 意思是传递一个拷贝，一个副本。
对于非引用类型：int、string、struct 地址变化，值不可改
对于引用类型：map、slice、chan、func, pointer 指针地址变化，但是值地址不变，可通过指针改

slice, array

从表现形式上来看

  1. 类型不同
    array是值类型，可类比于有序的struct，传递的是整个数组的拷贝
    slice是引用类型，又被称为动态的数组，传递的是一个的指向数组切片的指针

  2. 定义方式不同
    array需要在定义时指定长度[10]int, […]int，长度本身也是数据结构的一部分
    slice不需要指定长度，可变长的数组，在容量不足时可以自动扩容

  3. 底层实现不同
    array跟其他语言的实现都是一样的：一段连续的内存空间，每个元素有唯一索引来访问，由于内存连续，cpu可以很快的计算索引，快速迭代数组里面的所有元素
    但是使用方式不一样，例如与c语言相比，C语言的数组变量是指向第一个数组元素的指针，而go的数组变量是一个值，不可分割。 – 那应该是不能像c语言那样通过指针++来访问其中的元素的吧
    slice是对数组的抽象，它的数据结构中有一个是指向 array 的指针

new, make

都是用来分配内存的内建函数，分配在堆上，通过GC回收

  • make, 即分配，也初始化数据结构
  • new, 分配内存，返回指向内存的指针
    https://draveness.me/golang/docs/part2-foundation/ch05-keyword/golang-make-and-new/
    https://segmentfault.com/a/1190000018967149#:~:text=make%E5%92%8Cnew%E9%83%BD%E6%98%AF,%E6%98%AF%E6%8C%87%E5%90%91%E7%B1%BB%E5%9E%8B%E7%9A%84%E6%8C%87%E9%92%88%E3%80%82

对于一个已经存在变量，可对其指针进行赋值
如果不是已经存在的变量会如何呢？能对其直接赋值吗？

  • 不能
    panic: runtime error: invalid memory address or nil pointer dereference
    通过Go提供了new来初始化这一地址就可以解决。 对于普通类型，var后是nil, new不仅会返回一个指针，还会初始化指针指向的内容。所以之后可以直接通过指针赋值了。但对于引用类型不可以，new以后还是不可以赋值，因为new之后内容还是nil。
    but:
    主要区别在于引用类型。go会给引用类型初始化为nil，nil是不能直接赋值的。不能用new分配内存，无法直接赋值。
    make 不仅可以开辟内存，还可以给内存的类型初始化为其默认值。

why we need kafka

Posted on 2021-04-05

data powers the development of enterprise.
consumers views products on the web page -> collect infos -> recommend products

背景

数据是非常重要的，而且不能是片面的数据，而是全面的数据。
所以需要数据管道，这样一个关键性的组件，将来自各个不同地方的数据，以最高效的方式聚集在一起。

场景

消息队列，进程间通道
-> 出现多对多, 节点间连接混乱
-> 独立中间件, 提供应用程序接收来自其他应用程序的度量指标，并为其他系统提供查询服务器，即独立的队列系统
-> 业务场景多，如用户页面指标，活动指标，日志，监控(数据库)等。出现多个队列系统
-> 集成多个消息系统为单一的集中式的队列系统，可以发布通用型的数据，可以随业务数据增长而增长。

因为以上问题，kafka出现了

kafka是一款基于发布与订阅的消息系统，用于保存所有事务的持久化记录，可以按需读取，可以用于通过重放日志重建系统状态。
具有数据故障保护能力和性能伸缩能力。

Is go an object-oriented language?

Posted on 2021-01-11 Edited on 2021-04-05

According to official documents, this is one of the “Frequently Asked Questions(FAQ)”. And it’s what the answer is in the documents below.

I divided it into 5 points.

  1. Although go has types and methods and allows an object-oriented style of programming. there is no type hierarchy.

  2. The concept of “interface” in go provides a different approach that we believe is easy to use and in some ways more general.

  3. There are also ways to embed types in other types to provides something analogous, but not identical, to subclassing.

  4. Moreover, methods in go are more general than in C++ or Java: they can be defined for any sort of data, even built-in types, such as plain, “unboxed” integers. They are not restricted to structs(classes).

  5. Also the lack of an type hierarchy, makes “objects” in go feel much more lightweight than in languages such as C++ or Java.

In Short:

  1. 没有继承，但可以实现子类，使得对象更加轻量
  2. 有interface, 并且可以给所有数据类型定义方法，而C++和Java只能给结构体和类定义方法。

ref: https://golang.org/doc/faq
key-word: object-oriented
