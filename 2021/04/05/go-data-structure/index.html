Posted on 2021-04-05 Edited on 2021-04-13

什么是数据结构

数据类型的出现，是为了把数据分成所需内存大小不同的数据进行存储。

go 的几个关键数字

25 个关键字
3 个基本数据类型

  • bool(false), num(0), string(“”)
    8 个复合数据类型(除了struct, default: nil)
  • pointer, array, slice, map, channel, struct, func, interface

值类型

基本数据类型，int,float,bool,string, 以及数组和 struct 特点：变量直接存储值，内存通常在栈中分配，栈在函数调用完会被释放

引用类型

slice, map, channel, pointer, func, interface
是一种结构体+元素指针的混合类型

基本数据类型

bool, num, string
num:

  • int, int8, int16, int32, int64
  • uint, uint8, uint16, uint32, unit64, uintptr
  • byte // uint8
  • rune // int32 for unicode
  • float32, float64
  • complex64, complex32

int

go 中的 int 是 int 64

由上，因为a是int64，所以不同存储空间间隔8bytes=64bits
不仅如此还可以证明，在go语言中，参数传递，传递的是“值”，而不是引用。
“值” 意思是传递一个拷贝，一个副本。
对于非引用类型：int、string、struct 地址变化，值不可改
对于引用类型：map、slice、chan、func, pointer 指针地址变化，但是值地址不变，可通过指针改

string, byte, rune

utf8, unicode

https://www.zhihu.com/question/23374078

unicode 是字符集，16进制范围 0000-FFFF, 0～65536
utf-8是一种编码方式，一种转换规则，将字符集变成计算机可以理解的二进制序列, 顾名思义，是一套以8位为一个编码单位的可变长编码。

每一个16进制数，表示成一个四位的二进制数
16^4 = 2^4^4 = 2^16
unicode 用16位二进制代表一个符号

问题：
英文只有8bit, 所以所有字符都用16bit存放浪费空间
所以一些编码规则出现，为了更合理高效的利用内存

规则为：

string是一个字节 []uint8 , rune是4个字节 []int32 (正好对应最大的utf8)
所以rune可以直接将[]byte解析成对应的unicode字符

中文的特殊性：

内存地址是最小单位是 1byte, 也就是打印出来的16进制指针的单位是byte
string 底层实现是 byte 数组，所以长度是字节长度
1byte 是 8bit, 每一个string是一个unit8, 也就是 1byte
但是

go 为每一个 string 类型分配 16byte 内存

slice, array

从表现形式上来看

  1. 类型不同
    array是值类型，可类比于有序的struct，传递的是整个数组的拷贝
    slice是引用类型，又被称为动态的数组，传递的是一个的指向数组切片的指针

  2. 定义方式不同
    array需要在定义时指定长度[10]int, […]int，长度本身也是数据结构的一部分
    slice不需要指定长度，可变长的数组，在容量不足时可以自动扩容

  3. 底层实现不同
    array跟其他语言的实现都是一样的：一段连续的内存空间，每个元素有唯一索引来访问，由于内存连续，cpu可以很快的计算索引，快速迭代数组里面的所有元素
    但是使用方式不一样，例如与c语言相比，C语言的数组变量是指向第一个数组元素的指针，而go的数组变量是一个值，不可分割。 – 那应该是不能像c语言那样通过指针++来访问其中的元素的吧
    slice是对数组的抽象，它的数据结构中有一个是指向 array 的指针

new, make

都是用来分配内存的内建函数，分配在堆上，通过GC回收

  • make, 即分配，也初始化数据结构
  • new, 分配内存，返回指向内存的指针
    https://draveness.me/golang/docs/part2-foundation/ch05-keyword/golang-make-and-new/
    https://segmentfault.com/a/1190000018967149#:~:text=make%E5%92%8Cnew%E9%83%BD%E6%98%AF,%E6%98%AF%E6%8C%87%E5%90%91%E7%B1%BB%E5%9E%8B%E7%9A%84%E6%8C%87%E9%92%88%E3%80%82

对于一个已经存在变量，可对其指针进行赋值
如果不是已经存在的变量会如何呢？能对其直接赋值吗？

  • 不能
    panic: runtime error: invalid memory address or nil pointer dereference
    通过Go提供了new来初始化这一地址就可以解决。 对于普通类型，var后是nil, new不仅会返回一个指针，还会初始化指针指向的内容。所以之后可以直接通过指针赋值了。但对于引用类型不可以，new以后还是不可以赋值，因为new之后内容还是nil。
    but:
    主要区别在于引用类型。go会给引用类型初始化为nil，nil是不能直接赋值的。不能用new分配内存，无法直接赋值。
    make 不仅可以开辟内存，还可以给内存的类型初始化为其默认值。
