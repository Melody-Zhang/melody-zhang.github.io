使用 sync.WaitGroup 和 channel

what is sync.WaitGroup for？

首先，它是thread-safe;
其次，它等待所有goroutines结束。
call Add(): 主goroutine添加需要等待的goroutine数;
call Wait(): 阻塞主goroutine直到所有的goroutine都结束;
call Done(): 每个goroutine结束的时候调用。

noted:
add不能设置为负数，否则panic
waitGroup有一个队列，队列有一个counter计数, add计数加1，done计数减1，wait block until counter=0，退出。

需要注意，如果不写defer s.Done()会导致
fatal error: deadlock!

go - knowledge

go 语言的独特之处

  1. go 对类型非常苛刻，不允许强制的类型转换，不能隐式的类型转换(即使是原有类型和别名)，可以显式的类型转换
  2. go 没有前置++
  3. 与 c,c++ 相比，go 的string 默认值是””, 而不是nil
  4. 不支持指针运算
  5. 不同长度数组之间不可用 “==” 进行比较
  6. 数组对应位置元素也要相等，才是true

main

  • 应用程序入口 cli.go - 不必须是 main.go
  • package main - must
  • func main(){} - must, and no args、no returns
  • args 通过 os.Args or flag
  • return 通过 os.Exit(0) 正常退出，os.Exit(non-zero) 异常退出，其中的数字即为应用程序的退出状态，或者main函数返回值
  • 可以通过build cli.go, echo $? 进行查看

panic 和 os.Exit()

panic - defer & stack info
os.Exit - no defer & no stack info

大小写

包名可以小写
通过大小写控制访问权限

  • 大写: 外部可访问
  • 小写：外部不可访问
    结构体中的字段名
  • 如果字段名，首字母小写, 该字段无法被外部包访问和解析，如json.Marshal()等
  • 结构体本身的命名，首字母小写，没有关系
    所以结构体中的字段名最好大写，即使是内部包访问

== 比较的是内容还是地址

according to what kind of type you compare

  1. 基础类型：int, bool, string - 直接比较值
  2. 复合类型：数组和结构体 - 逐字段比较，长度位置数据都要一致
  3. 引用类型：slice, map, channel, ptr, 除了slice 其他都比较引用
  4. 接口类型：interface，比较类型以及值
  5. slice: 不可以比较

为什么 slice 不可比较

  1. 因为slice可以引用自己，实现难度大
  2. slice如果跟array行为不一致，会使人感到混淆
  3. slice can only be compared to nil

slice, array

从表现形式上来看

  1. 类型不同
    array是值类型，可类比于有序的struct，传递的是整个数组的拷贝
    slice是引用类型，又被称为动态的数组，传递的是一个的指向数组切片的指针

  2. 定义方式不同
    array需要在定义时指定长度[10]int, […]int，长度本身也是数据结构的一部分
    slice不需要指定长度，可变长的数组，在容量不足时可以自动扩容

  3. 底层实现不同
    array跟其他语言的实现都是一样的：一段连续的内存空间，每个元素有唯一索引来访问，由于内存连续，cpu可以很快的计算索引，快速迭代数组里面的所有元素
    但是使用方式不一样，例如与c语言相比，C语言的数组变量是指向第一个数组元素的指针，而go的数组变量是一个值，不可分割。 – 那应该是不能像c语言那样通过指针++来访问其中的元素的吧
    slice是对数组的抽象，它的数据结构中有一个是指向 array 的指针

对于一个已经存在变量，可对其指针进行赋值
如果不是已经存在的变量会如何呢？能对其直接赋值吗？

  • 不能
    panic: runtime error: invalid memory address or nil pointer dereference
    通过Go提供了new来初始化这一地址就可以解决。 对于普通类型，var后是nil, new不仅会返回一个指针，还会初始化指针指向的内容。所以之后可以直接通过指针赋值了。但对于引用类型不可以，new以后还是不可以赋值，因为new之后内容还是nil。
    but:
    主要区别在于引用类型。go会给引用类型初始化为nil，nil是不能直接赋值的。不能用new分配内存，无法直接赋值。
    make 不仅可以开辟内存，还可以给内存的类型初始化为其默认值。

why we need kafka

Posted on 2021-04-05

data powers the development of enterprise.
consumers views products on the web page -> collect infos -> recommend products

背景

数据是非常重要的，而且不能是片面的数据，而是全面的数据。
所以需要数据管道，这样一个关键性的组件，将来自各个不同地方的数据，以最高效的方式聚集在一起。

场景

消息队列，进程间通道
-> 出现多对多, 节点间连接混乱
-> 独立中间件, 提供应用程序接收来自其他应用程序的度量指标，并为其他系统提供查询服务器，即独立的队列系统
-> 业务场景多，如用户页面指标，活动指标，日志，监控(数据库)等。出现多个队列系统
-> 集成多个消息系统为单一的集中式的队列系统，可以发布通用型的数据，可以随业务数据增长而增长。

因为以上问题，kafka出现了

kafka是一款基于发布与订阅的消息系统，用于保存所有事务的持久化记录，可以按需读取，可以用于通过重放日志重建系统状态。
具有数据故障保护能力和性能伸缩能力。
