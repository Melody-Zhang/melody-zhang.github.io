noted

  1. array的操作与slice有所不同，array是不可变的连续存储空间，所以增加和删除这种操作都是不被允许的，只能通过slice来操作
  2. 但array和python中的tuple不同，只是空间不可变，某个空间上的元素还是可以直接被赋值的。
  3. 实际上，array不需要自己实现的方法只有以下几种：len(), cap(), 遍历，根据索引获取元素值，根据索引改变元素值，指针运算，其中元素的指针运算
  4. 需要注意的是：append()方法的入参是slice, 所以对array的增，删，插入等操作可以借助slice, 但是append会在第一个入参的slice的地址上进行操作，也就是会在array原有的地址空间进行操作，看起来就像覆盖写，会改变原有的array。

go - 两个协程接替打印基数和偶数

Posted on 2021-04-11 Edited on 2021-04-18

go

使用 sync.WaitGroup 和 channel

what is sync.WaitGroup for？

首先，它是thread-safe;
其次，它等待所有goroutines结束。
call Add(): 主goroutine添加需要等待的goroutine数;
call Wait(): 阻塞主goroutine直到所有的goroutine都结束;
call Done(): 每个goroutine结束的时候调用。

noted:
add不能设置为负数，否则panic
waitGroup有一个队列，队列有一个counter计数, add计数加1，done计数减1，wait block until counter=0，退出。

需要注意，如果不写defer s.Done()会导致
fatal error: deadlock!

what is channel for?

使用没有buffer的channel的“消息没有接收成功就阻塞”的作用
使得两个打印函数的自增id:i保持一致。
