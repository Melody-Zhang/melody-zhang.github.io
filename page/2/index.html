拓扑排序

把一个图，排序成一个线性序列。
方法：类似于广度优先

  1. 建立图和入度表
  2. 先把入度为0的起始点加入队列
  3. 如果队列不为空，弹出队列中的第一个点，加入res
  4. 将与这个点相连的点的入度减一，如果为0，则加入队列
  5. 执行2，3直到队列为空
  6. 如果res的长度和点数相同，说明排序完成，否则说明图中有环

并查集

并查集，故名思义包含两个操作
一个是 “合并”： union(a,b) 将a所属集合和b所属集合合并
一个是“查询”： find(c) 查找c的祖先
所以如何判断a和b是否联通呢？if find(a)==find(b)
通过使用并查集，我们可以将一堆点分成相互连通的不同的几个集合。

函数的实现以及优化

有两个数据结构 id 和 size
初始化每一个点的父节点是自己id[i]=i，集合个数是1 size[i]=1
find 做的事情是找到父节点，顺便路径压缩以增加查找速度

union 做的事情是找到两个点的父节点，将小的集合和并入大的集合(俗称按秩合并)

MST 最小生成树

什么是联通图的生成树

一个连通图的生成树是一个极小的连通子图，它包含图中全部的n个顶点，但只有构成一棵树的n-1条边。

有两种思路找最小生成树

整体是贪心算法的思想
实现
Prim’s Tree - 优先队列，树的方法，从一个点开始覆盖其他点，优先队列选择最小消耗
也就是从任意一点开始，每次加入的点都是离它最近的，直到覆盖所有点
kruskal’s Forest - 并差集，森林的方法，从一棵树开始覆盖其他的边，排序后使用并查集
在森林中，每次找最近的两颗树相连

时间复杂度分析

Elog(V) E是边的个数，V是点的个数

kruskal实现

A是一个挑选出来的边的集合，先对所有边按权重排序O(ElogE)，然后依次遍历这些边，如果边的两个顶点不属于同一颗树，那么合并两颗树(ElogV)，挑出这条边，如果属于同一颗树则跳过。

dijkstra 单源最短路径算法

计算从起点出发，到每个点的最短距离
算法思路：
每次从”未求出最短路径的点”中取出“距离起点最近的点”，以这个点为桥梁刷新“未求出最短路径的点的”距离
也就是记录每个点到起点的最短距离
流程：

  1. 建立临接链表edges[i] = append(edges[i], edge{city, cost})
    建立初始dist[i] = math.MaxInt32
  2. 把起点放进优先队列
  3. 对于优先队列中的每一个点执行以下循环：
  • 弹出cost最小的加入结果队列，cost为d1
  • 遍历该点的所有临接，如果d1+v.cost<dist[i], 说明之前没有访问过，更新dist[i]和v.cost
  1. 如果len(结果队列)==点的总数，返回结果队列最后一个点的cost, 否则返回-1
